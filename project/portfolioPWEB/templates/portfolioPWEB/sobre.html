01

Arquitetura MVT

A aplicação segue a arquitetura MVT do Django: Model, View e Template. Esta organização separa os dados, a lógica e a apresentação visual da aplicação.

Desenho da arquitetura MVT

Model

Define a estrutura dos dados. Cada classe no ficheiro models.py representa uma tabela da base de dados.

View

Recebe os pedidos do utilizador, consulta os modelos, processa a lógica e envia dados para os templates.

Template

Define a apresentação visual. São ficheiros HTML que usam a linguagem template do Django para mostrar dados dinâmicos.

Fluxo de informação

1
Browser envia um pedido HTTP para um URL.
↓
2
urls.py associa esse URL a uma função view.
↓
3
View consulta os modelos e prepara os dados.
↓
4
Template recebe os dados e gera o HTML final.
↓
5
Resposta HTTP devolve a página ao browser.

Modelação

A modelação representa as entidades principais da aplicação e as relações entre elas. O objetivo é organizar o portfólio de forma relacional, ligando projetos, tecnologias, competências, formações e unidades curriculares.

Diagrama entidade-relação

Entidades e relações principais

Licenciatura

Relaciona-se com UnidadeCurricular.

UnidadeCurricular

Relaciona-se com Docente, Competência, Tecnologia e Projeto.

Projeto

Pertence a uma UC e pode ter várias tecnologias e competências.

Tecnologia

Pode estar associada a competências, UCs, projetos e TFCs.

Competência

Pode estar associada a tecnologias, projetos, formações e áreas de interesse.

MakingOf

Documenta o processo de desenvolvimento de um projeto.

Implementação em models.py

A modelação foi implementada no ficheiro models.py, usando campos como CharField, TextField, URLField, ImageField, ForeignKey e ManyToManyField. 

class Projeto(models.Model):
    titulo = models.CharField(max_length=50)
    descricao = models.TextField()
    conceitos_aplicados = models.TextField()
    tecnologias_resumo = models.TextField()
    ano = models.IntegerField()
    imagem = models.ImageField(upload_to='projetos/', blank=True, null=True)
    video_demo = models.URLField(blank=True, null=True)
    github = models.URLField(blank=True, null=True)

    uc = models.ForeignKey(
        UnidadeCurricular,
        on_delete=models.CASCADE,
        related_name='projetos'
    )

    tecnologias = models.ManyToManyField(
        Tecnologia,
        related_name='projetos',
        blank=True
    )

    competencias = models.ManyToManyField(
        Competencia,
        related_name='projetos',
        blank=True
    )
