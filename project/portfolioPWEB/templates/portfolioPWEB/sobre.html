Arquitetura, modelação e tecnologias

Arquitetura MVT

Como a aplicação Django está organizada

Esta aplicação segue a arquitetura MVT do Django: os Models representam os dados, as Views tratam a lógica da aplicação e os Templates apresentam a informação ao utilizador.

Model

Define entidades como Projeto, Tecnologia, Competência, Formação e Making Of.

View

Recebe pedidos, consulta a base de dados e envia dados para os templates.

Template

Constrói as páginas HTML visíveis no browser.

Modelação

Entidades principais

A aplicação está organizada em entidades relacionadas entre si. Os projetos podem estar associados a tecnologias, competências e unidades curriculares. As formações também podem estar ligadas a competências.

Tecnologias

Ferramentas usadas

  • Django para backend e organização MVT
  • Python para lógica da aplicação
  • HTML para estrutura das páginas
  • CSS para identidade visual e responsividade
  • SQLite/base de dados para guardar informação
  • GitHub para controlo de versões
